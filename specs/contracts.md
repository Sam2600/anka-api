# anka-api — Contracts Module Spec

## Models

### `Contract`
File: `app/Models/Contract.php`

**All fields:**
```
id                  UUID PK
tenant_id           UUID FK → tenants
deal_id             UUID FK → deals  (SET NULL on deal delete, nullable)
contract_number     varchar(255) UNIQUE  auto: CON-0001 via contract_number_seq
client              varchar(255)
total_value         decimal(14,2) default 0
revenue_recognized  decimal(14,2) default 0  incremented by invoice payments
status              enum  Draft | Active | Completed | Cancelled  default: Draft
start_date          date  nullable
end_date            date  nullable
notes               text  nullable
created_at, updated_at, deleted_at
```

**Relationships:**
```
belongsTo: Deal           (FK: deal_id)
hasOne:    Project        (UNIQUE FK: projects.contract_id)
hasMany:   Milestone
hasMany:   Invoice
```

### `Milestone`
File: `app/Models/Milestone.php`
```
id            UUID PK
tenant_id     UUID FK → tenants
contract_id   UUID FK → contracts
name          varchar(255)
due_date      date
amount        decimal(14,2)
status        enum  Pending | In Progress | Completed  default: Pending
completed_at  timestamp  nullable
```
No soft deletes.

### `Invoice`
File: `app/Models/Invoice.php`
```
id              UUID PK
tenant_id       UUID FK → tenants
contract_id     UUID FK → contracts  (RESTRICT delete)
milestone_id    UUID FK → milestones  (SET NULL, nullable)
invoice_number  varchar(255) UNIQUE  auto: INV-1042 via invoice_number_seq
issue_date      date
due_date        date  nullable
amount          decimal(14,2)
tax             decimal(14,2)
total           decimal(14,2)  GENERATED ALWAYS AS (amount + tax) STORED — never set from PHP
status          enum  Draft | Pending | Paid | Overdue | Cancelled
paid_at         timestamp  nullable
notes           text  nullable
created_at, updated_at, deleted_at
```

## Contract Status Values and Transitions

| Status | Meaning |
|---|---|
| `Draft` | Created by win_deal(); not yet active |
| `Active` | Contract in force |
| `Completed` | Delivery finished |
| `Cancelled` | Contract terminated |

**Transitions:** Any status can be set to any other via `PATCH /contracts/{contract}` — there is no enforced transition guard on the contract status itself.

⚠️ **MISSING:** No `Signed` status. The desired flow "Contract Signed → Create Project" is not implemented as a distinct status gate. Projects are created simultaneously with Contracts inside `win_deal()`.

## Invoice Status Values

| Status | Set By |
|---|---|
| `Draft` | Manual (create with status Draft) |
| `Pending` | Manual (create with status Pending) |
| `Paid` | `PATCH /invoices/{invoice}/pay` |
| `Overdue` | Computed client-side (Pending + due_date past) — not a DB value |
| `Cancelled` | Manual |

## Business Rules

### Contract
- Contracts are created **exclusively** by the `win_deal()` stored procedure when a Deal is won.
- No `POST /contracts` endpoint exists — do not add one without migrating the SP logic.
- `contract_number` is auto-generated by the PostgreSQL sequence — never set it manually.
- `revenue_recognized` is incremented atomically when an Invoice is paid — never set it directly.
- Updatable fields via `PATCH`: `status`, `notes`, `end_date`, `total_value`.

### Invoice
- `total` is a PostgreSQL GENERATED column — never include it in PHP writes.
- `invoice_number` is auto-generated by sequence — never set it manually.
- Paying an invoice: `PATCH /invoices/{invoice}/pay` runs in a `DB::transaction()` that simultaneously:
  1. Sets `invoices.status = 'Paid'` and `paid_at = now()`
  2. Increments `contracts.revenue_recognized` by `invoice.total`
- Invoice can only be deleted — there is no update endpoint.

### Milestone
- Full CRUD (create, update, delete) — no special guards.
- When a milestone is linked to an invoice, deleting the milestone sets `invoices.milestone_id = NULL`.

## API Endpoints

### Contracts

#### `GET /api/contracts`
- Filter: `deal_id`, `status`
- Pagination: `per_page` (default 50, max 200)
- Response: paginated `ContractResource`

#### `GET /api/contracts/{contract}`
- Response: `ContractResource`

#### `PATCH /api/contracts/{contract}`
- Allowed updates: `status` (enum), `notes`, `end_date`, `total_value`
- Response: `ContractResource`

#### `DELETE /api/contracts/{contract}`
- Soft delete
- Response: 204

#### `GET /api/contracts/{contract}/project`
- Returns linked project or `{ data: null }`
- Response: `ProjectResource` or `{ data: null }`

### Invoices

#### `GET /api/invoices`
- Filter: `contract_id`, `status`
- Pagination: `per_page` (default 50, max 200)

#### `POST /api/invoices`
- Required: `contract_id`, `issue_date`, `amount`
- Optional: `milestone_id`, `due_date`, `tax`, `notes`
- `invoice_number` auto-assigned by DB sequence
- Response: `InvoiceResource`

#### `GET /api/invoices/{invoice}`

#### `DELETE /api/invoices/{invoice}`
- Soft delete; 204

#### `PATCH /api/invoices/{invoice}/pay`
- No request body needed
- Transaction: marks Paid + increments contract.revenue_recognized
- Response: `InvoiceResource`

### Milestones

#### `GET /api/milestones`
#### `POST /api/milestones`
Required: `contract_id`, `name`, `due_date`, `amount`
#### `GET /api/milestones/{milestone}`
#### `PUT /api/milestones/{milestone}`
#### `DELETE /api/milestones/{milestone}`
Response: 204
